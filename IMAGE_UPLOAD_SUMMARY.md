## ⚙️ Setup Required

### Supabase Storage Bucket

**IMPORTANT**: You need to create the storage bucket first!

1. Go to Supabase Dashboard → Storage
2. Create bucket named `products`
3. Enable **public access**
4. Set up policies (see `STORAGE_BUCKET_SETUP.md`)

**Quick SQL Setup:**
```sql
-- Run in Supabase SQL Editor
CREATE POLICY "Authenticated users can upload images"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (bucket_id = 'products');

CREATE POLICY "Public can view images"
ON storage.objects FOR SELECT
TO public
USING (bucket_id = 'products');
```

## 📊 Technical Details

### Upload Process:
1. User selects/drops image
2. Validate file type (must be image)
3. Validate file size (max 5MB)
4. Generate unique filename
5. Upload to Supabase Storage
6. Get public URL
7. Update form data
8. Show preview

### File Naming:
- Format: `{timestamp}-{random}.{extension}`
- Example: `1704123456789-abc123.jpg`
- Ensures unique filenames
- Prevents overwrites

### Storage Location:
- Bucket: `products`
- Public: Yes
- URL: Auto-generated by Supabase

## 🐛 Common Issues

### Upload Fails
**Cause**: Bucket doesn't exist or not public  
**Fix**: Create bucket and enable public access

### Image Not Showing
**Cause**: Bucket policies missing  
**Fix**: Add SELECT policy for public access

### "Permission Denied"
**Cause**: User not authenticated  
**Fix**: Ensure user is logged in as admin
